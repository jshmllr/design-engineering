# Week 3: Systems, accessibility, and performance

## Objectives
- Translate a UI element into a reusable system component.
- Apply accessibility, theming, and performance thinking.

## Daily plan
**Day 11 — Systemize a component**
- Refactor the prototype into a reusable component with variants.

**Day 12 — Accessibility pass**
- Add keyboard support and ARIA where appropriate.
- Check contrast and focus states.

**Day 13 — Theming or responsive behavior**
- Add tokens or variables for theming.
- Validate responsiveness across layouts.

**Day 14 — Performance reasoning**
- Identify potential performance pitfalls (reflow, large animations, etc.).
- Document mitigation steps.

**Day 15 — Mini case study**
- Write a short case study outlining goals, implementation, and learnings.

## Outputs
- Reusable component
- Accessibility checklist
- Mini case study
