# Week 4: Product thinking & storytelling

## Objectives
- Build a small feature concept with a clear user problem.
- Communicate outcomes with a polished write-up.

## Daily plan
**Day 16 — Define the problem**
- Write a short problem statement and target user.

**Day 17 — Build core flow**
- Implement the primary interaction flow.

**Day 18 — UI and motion refinement**
- Add interaction polish and visual clarity.

**Day 19 — Document UX goals**
- Explain constraints, UX goals, and tradeoffs.

**Day 20 — Ship and share**
- Publish the prototype and write-up.

**Day 21 — Reflection**
- Capture what you learned and what you would change next.

**Day 22 — Optional walkthrough**
- Record a short demo walkthrough.

**Day 23–30 — Consolidate**
- Organize artifacts, update README, and outline next steps.

## Outputs
- Feature prototype
- Final write-up
- Reflection
